 * ================================================================================
 * OBJECT-ORIENTED PROGRAMMING CONCEPTS DEMONSTRATED
 * ================================================================================
 * 
 * 1. CLASS & OBJECT
 *    ---------------------------------------------------------------------------
 *    Definition: A class is a blueprint for creating objects
 *    
 *    Implementation:
 *    - Movie class: Represents individual movie entities
 *    - MovieDatabase class: Manages collection of movies
 *    - Objects created: database (MovieDatabase), individual movies (Movie)
 * 
 * 2. ENCAPSULATION
 *    ---------------------------------------------------------------------------
 *    Definition: Bundling data and methods, hiding internal details
 *    
 *    Implementation:
 *    - Private data members: name, id, year, language, rating
 *    - Public interface: getters, setters, and utility methods
 *    - Data protection: Cannot directly access movie.name or movie.rating
 *    - Must use: movie.getName() or movie.getRating()
 * 
 *    Benefits:
 *    - Data validation (rating kept within 1-5 range)
 *    - Controlled access to internal state
 *    - Flexibility to change implementation without affecting users
 * 
 * 3. ABSTRACTION
 *    ---------------------------------------------------------------------------
 *    Definition: Hiding complex implementation, exposing simple interface
 *    
 *    Implementation:
 *    - displayTopRatedMovies(): User doesn't see the search algorithm
 *    - displayMoviesByLanguage(): Filtering logic is hidden
 *    - displayLatestMovies(): Comparison logic abstracted away
 *    
 *    Benefits:
 *    - Simple to use: database.displayTopRatedMovies()
 *    - Complex logic hidden: loop, comparison, formatting
 *    - Easy to maintain and modify
 * 
 * 4. COMPOSITION
 *    ---------------------------------------------------------------------------
 *    Definition: "Has-a" relationship between classes
 *    
 *    Implementation:
 *    - MovieDatabase HAS-A array of Movie objects
 *    - movies[MAX_MOVIES]: Array member variable
 *    
 *    Relationship:
 *    MovieDatabase ??[contains]??> Movie[20]
 * 
 * 5. CONSTRUCTOR
 *    ---------------------------------------------------------------------------
 *    Definition: Special method to initialize objects
 *    
 *    Implementation:
 *    - Movie(): Default constructor (empty initialization)
 *    - Movie(params): Parameterized constructor (direct initialization)
 *    - MovieDatabase(): Initializes movieCount to 0
 *    
 *    Usage:
 *    Movie m1;  // Default constructor
 *    Movie m2("Inception", 1, 2010, "English", 5);  // Parameterized
 * 
 * 6. CONST CORRECTNESS
 *    ---------------------------------------------------------------------------
 *    Definition: Using 'const' to prevent unintended modifications
 *    
 *    Implementation:
 *    - const methods: getName() const, getRating() const
 *    - Guarantees: These methods won't modify the object
 *    - const parameters: void setName(const std::string& name)
 *    
 *    Benefits:
 *    - Compiler enforces data integrity
 *    - Clear intent in code
 *    - Prevents accidental modifications
 * 
 * ================================================================================
 * ALGORITHM EXPLANATIONS
 * ================================================================================
 * 
 * 1. Finding Top-Rated Movies
 *    ---------------------------------------------------------------------------
 *    Algorithm: Linear search with maximum tracking
 *    
 *    Step 1: Find maximum rating
 *      - Initialize maxRating = 0
 *      - Loop through all movies
 *      - Update maxRating if current movie rating is higher
 *    
 *    Step 2: Display all movies with maximum rating
 *      - Loop through all movies again
 *      - Display movie if rating equals maxRating
 *    
 *    Time Complexity: O(n) where n = number of movies
 * 
 * 2. Filtering by Language
 *    ---------------------------------------------------------------------------
 *    Algorithm: Linear search with string comparison
 *    
 *    - Loop through all movies
 *    - Compare movie.language with target language
 *    - Display if match found
 *    - Count total matches
 *    
 *    Time Complexity: O(n)
 * 
 * 3. Finding Latest Movies
 *    ---------------------------------------------------------------------------
 *    Algorithm: Similar to top-rated, but with year comparison
 *    
 *    Step 1: Find latest year
 *      - Initialize latestYear = 0
 *      - Loop and track maximum year
 *    
 *    Step 2: Display all movies from latest year
 *      - Loop and compare year with latestYear
 *      - Display matches
 *    
 *    Time Complexity: O(n)
 * 
 * ================================================================================
 * CODE ORGANIZATION
 * ================================================================================
 * 
 * File Structure:
 * 
 * Movie.h              - Movie class declaration (header)
 * Movie.cpp            - Movie class implementation
 * MovieDatabase.h      - MovieDatabase class declaration
 * MovieDatabase.cpp    - MovieDatabase class implementation
 * main.cpp             - Main program and demonstration
 * 
 * Design Pattern: Separation of Interface and Implementation
 * - .h files: Declarations (what the class can do)
 * - .cpp files: Definitions (how it does it)
 * 
 * Benefits:
 * - Modularity: Each class in separate files
 * - Maintainability: Easy to locate and modify code
 * - Reusability: Classes can be used in other projects
 * - Compilation efficiency: Changes to .cpp don't require recompiling everything
