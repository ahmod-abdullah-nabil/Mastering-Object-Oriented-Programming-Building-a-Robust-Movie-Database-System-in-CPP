?? TIPS FOR EACH FILE TYPE
???????????????????????????????????????????????????????????????????????????????

When reading .h files (headers):
  • Focus on the class structure
  • Notice what's private vs public
  • Understand method names and parameters
  • Don't worry about implementation yet

When reading .cpp files (implementation):
  • Look at constructors first
  • Understand getter/setter logic
  • Study algorithm implementations
  • Notice how methods use other methods

When reading .txt files (documentation):
  • Read straight through first time
  • Take notes on important concepts
  • Come back for reference later
  • Print if it helps you learn

When reading .md files (markdown):
  • Best viewed in Markdown viewer
  • Can be read as plain text too
  • Check the formatting in a viewer
  • Great for GitHub if you upload
